Objectif :
Analyser un texte brut bruité et extraire uniquement les informations SÛRES décrivant les paramètres du fichier à uploader (image, audio et vidéo uniquement).
Si une information n’est pas clairement mentionnée, ou si elle provient d’un élément non fiable (comme un nom de fichier ou une conversation), mets "not found".

Description des champs :
- accepted_source : type de fichier explicitement mentionné (ex : "JPG", "PNG", "MP4", "PDF", "image/*", "audio/*").
⚠️ N’infère jamais à partir d’un nom de fichier.
- file_size_limit : taille maximale autorisée (ex : "5 Mo", "10 MB", "not found")
- height_width : informations sur la taille/hauteur maximale et minimal (ex : "hauteur minimal de 300 px, largeur maximale de 1200 px", "not found")
- aspect_ratio : ratio d’image demandé (ex : "16:9", "1:1", "not found")
- other_file_infos : toute autre contrainte explicite sur le contenu ou la qualité du fichier (ex : "Must be landscape", "Transparent background required").
⚠️ N’ajoute pas de texte qui ne correspond pas à une contrainte (comme un nom de fichier, un numéro de téléphone, une date, ou un message).

Règles :
- ❌ Ignore totalement les noms de fichiers, dates, heures, messages, numéros, ou phrases hors contexte.
- ❌ Ne remplis jamais un champ à partir d’un mot ou d’un exemple isolé sans lien explicite avec une règle.
- ✅ Si plusieurs valeurs sont mentionnées, garde la plus précise.
- ✅ Si aucune contrainte claire n’est trouvée, remplis le champ avec "not found".

Exemple 1 :
Texte : "Le fichier doit être une image JPG ou PNG de 2000x1000 max et 5 Mo maximum."
Réponse :
{
    "accepted_source": "jpg, png",
    "file_size_limit": "5 Mo",
    "height_width": "max width 2000px, max height 1000px",
    "aspect_ratio": "not found",
    "other_file_infos": "not found"
}
